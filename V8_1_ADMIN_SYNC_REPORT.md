# V8.1 Admin Sync & Usability Pass

## Что доработано

1. Добавлена публичная проверка версии CMS: `/api/cms/version`.
2. Добавлен клиентский компонент `CmsAutoRefresh`, который на публичных страницах проверяет опубликованную CMS-версию каждые 12 секунд.
3. Если админ публикует изменения, открытые публичные страницы на других устройствах мягко обновляют данные через `router.refresh()` без ручного перезапуска сервера.
4. Админ-раздел получил понятное объяснение разницы между черновиком и публикацией.
5. В обзоре админки добавлен блок “Как публикация попадает на все устройства”.
6. После публикации админка показывает корректное сообщение: изменения появятся на открытых устройствах автоматически в течение 12 секунд или сразу после обновления страницы.
7. API `/api/admin/cms` получил no-store заголовки, чтобы админка не ловила старые данные из кеша.
8. Публичная CMS-версия также отдаётся с no-store заголовками.
9. После публикации админка вызывает `router.refresh()`, чтобы сама панель тоже не висела на старом состоянии.

## Важное уточнение по “всем устройствам”

Изменения будут видны всем устройствам, которые открывают один и тот же запущенный сайт через один сервер/домен. Если сайт запущен только локально на ноутбуке, другие устройства должны открывать именно этот локальный сервер по сетевому адресу. Для настоящего продакшена нужен общий сервер/хостинг и PostgreSQL.
