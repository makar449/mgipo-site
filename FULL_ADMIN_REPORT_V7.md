# V7 Full Admin Control

## Что изменено

- Админ-панель больше не ограничена редактированием нескольких текстов главной страницы.
- Добавлена единая CMS-структура сайта: опубликованная версия + черновик.
- После публикации изменения сохраняются на сервере в `.local-data/admin-cms.json` и сразу используются публичным сайтом.
- Изменения видят все устройства, которые открывают этот же локально запущенный сайт.

## Разделы админ-панели

1. Обзор — состояние сайта, количество программ, заявок и документов.
2. Заявки — данные пользователя, тема, комментарий, статус, приоритет, заметка менеджера, CSV.
3. Главная — hero, CTA, описание, блок организаций, статистика.
4. Компания — короткое/полное название, подпись логотипа, телефон, email, адрес, часы, лицензия.
5. Шапка и футер — пункты меню, видимость, подписи, ссылки, описание футера, копирайт.
6. Программы — поиск по 450 программам, редактирование названия, категории, статуса, цены, описаний, модулей, кому подходит, что входит, преимуществ, FAQ, ключевых слов и вариантов обучения.
7. Документы — правовые документы, названия, описания, ссылки и видимость.
8. SEO — title, description, ключевые фразы, OG-title, OG-description.
9. Стиль — базовые дизайн-токены: основной цвет, фон, цвет текста, плотность, радиусы.

## Что подключено к публичному сайту

- Header берёт название, подпись и меню из опубликованной CMS.
- Footer берёт контакты, описание, копирайт и ссылки из опубликованной CMS.
- Главная берёт hero, CTA, статистику и блок организаций из опубликованной CMS.
- Каталог программ берёт опубликованный список программ из CMS.
- Страницы программ берут опубликованные данные программ из CMS.
- Формы заявок используют опубликованный список программ из CMS.
- Контакты используют опубликованные данные компании из CMS.
- Документы используют опубликованные документы и лицензионную информацию из CMS.
- Цены используют опубликованные программы из CMS.

## Проверки

- `npm run lint` — без ошибок.
- `npm run check:content` — без ошибок.
- `npx tsc --noEmit` в среде не может завершиться из-за отсутствия сгенерированного Prisma Client: `@prisma/client` не содержит `PrismaClient`. После локального `npx prisma generate` эта часть должна решиться.
