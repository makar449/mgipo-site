# V8.2 Admin Working UI & Overflow Hardening

Что исправлено в этой версии:

1. Исправлена Prisma-схема: в `ServiceVariant` больше нет дублирующего поля `price`, из-за которого схема была невалидной.
2. Усилен глобальный no-overflow слой: тексты, карточки, формы, кнопки, таблицы, админка и публичные блоки больше не должны вылезать за рамки.
3. Базовая кнопка больше не заставляет весь текст быть в одну строку. Header сохраняет nowrap только для меню, а остальные кнопки могут безопасно переноситься на маленьких экранах.
4. Админка стала рабочее и понятнее:
   - отображается состояние несохранённых изменений;
   - есть предупреждение при закрытии страницы с несохранённым черновиком;
   - добавлено понятное объяснение: черновик / публикация / обновление на устройствах;
   - можно создавать программу;
   - можно копировать выбранную программу;
   - можно скрывать программу;
   - можно добавлять пункты меню;
   - можно удалять пункты меню;
   - можно добавлять колонки футера;
   - можно добавлять и удалять ссылки футера;
   - можно добавлять и удалять документы;
   - сохранение черновика и публикация по-прежнему идут через общий CMS-слой.
5. Публичный сайт и админка продолжают работать через общий источник данных. После публикации изменения видны всем устройствам, которые открывают этот же сервер/домен.
6. Обновлены статические QA-проверки, чтобы ловить повторение старых проблем: дубли в Prisma, принудительный nowrap кнопок, отсутствие админских действий и отсутствие overflow-hardening CSS.

Проверки, выполненные в среде:

- `npm run lint` — без ошибок.
- `npm run check:content` — успешно.
- `npm run test:unit` — успешно.
- `npm run test:e2e:smoke` — успешно.
- `npm run test:a11y` — успешно.
- `npm run test:visual` — успешно.

Ограничение среды:

- `prisma validate`, `prisma generate`, `typecheck` и `build` не завершаются в этой среде из-за невозможности скачать Prisma engine с `binaries.prisma.sh`. Сама Prisma-схема вручную исправлена от найденной критической ошибки.
