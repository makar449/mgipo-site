# V8.5 Site-wide QA pass

Исправления сделаны не точечно по одному блоку, а по всему UI-слою сайта.

## Что изменено

- Убраны глобальные правила `overflow-wrap:anywhere`, которые ломали русские слова по буквам.
- Убрана автоматическая расстановка переносов `hyphens:auto`, из-за которой заголовки и карточки резались по слогам.
- Убраны `break-words` из карточек программ.
- Статистика на главной больше не принудительно строится в четыре узкие колонки внутри hero-текста.
- Статистические подписи сокращены до нормальных деловых формулировок.
- Добавлены финальные CSS-правила v8.5 для нормального переноса слов, карточек, формы, админки, кнопок, hero и мобильных экранов.
- 3D-блок ограничен по высоте и ширине, чтобы не создавать пустые провалы и не перекрывать кнопки.
- Добавлен static visual QA, который запрещает возвращать `overflow-wrap:anywhere`, `hyphens:auto`, `break-words` и четыре узкие колонки статистики.

## Проверки

- `npm run lint`
- `npm run check:content`
- `npm run test:unit`
- `npm run test:e2e:smoke`
- `npm run test:a11y`
- `npm run test:visual`

## Что важно

Если в админке вручную ввести очень длинное слово без пробелов, любой сайт может столкнуться с переполнением. Для нормального русского текста с пробелами перенос теперь идёт по словам, а не по буквам/слогам.
